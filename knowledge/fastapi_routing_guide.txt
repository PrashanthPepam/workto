FastAPI Routing Guide

FastAPI uses path operation decorators to define routes.

Basic route definition:
    from fastapi import FastAPI
    app = FastAPI()

    @app.get("/items/{item_id}")
    async def read_item(item_id: int, q: str | None = None):
        return {"item_id": item_id, "q": q}

Path parameters are declared in the path string and matched to function arguments.
Query parameters are function arguments not in the path string.

Request body (POST/PUT):
    from pydantic import BaseModel

    class Item(BaseModel):
        name: str
        price: float

    @app.post("/items/")
    async def create_item(item: Item):
        return item

Routers (APIRouter) help split large apps into modules:
    from fastapi import APIRouter
    router = APIRouter(prefix="/chats", tags=["chats"])

    @router.get("/")
    async def list_chats():
        ...

    # In main.py:
    app.include_router(router)

Status codes and error handling:
    from fastapi import HTTPException
    raise HTTPException(status_code=404, detail="Chat not found")

Response models enforce output schema:
    @app.get("/items/{id}", response_model=ItemResponse)
